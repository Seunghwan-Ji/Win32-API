윈도우 API에서 SAL(Source Annotation Language)은 Microsoft에서 제공하는 주석 기반 언어로,   
코드의 안전성과 정확성을 검증하기 위해 사용됩니다.   
SAL은 함수 매개변수, 반환값, 메모리 상태 등을 명시적으로 설명하여 컴파일러와 정적 분석 도구가   
코드를 더 잘 이해하고 잠재적인 오류를 찾아낼 수 있도록 돕습니다.   

---
### SAL의 주요 목적
1. 코드 분석: 정적 분석 도구가 함수 호출과 메모리 사용을 검증할 수 있도록 돕습니다.
2. 문서화: 함수와 그 매개변수의 역할을 명확히 하여 개발자 간의 이해를 높입니다.
3. 버그 방지: 잘못된 함수 호출, 버퍼 오버플로우, NULL 포인터 참조 등의 문제를 미리 방지합니다.

---
### SAL의 주요 키워드
SAL 키워드는 함수 매개변수나 반환값의 의도를 설명하는 주석으로 사용됩니다.
일부 주요 키워드는 다음과 같습니다:

1. 매개변수 주석
	- _In_: 함수가 읽기용으로 사용하는 매개변수임을 나타냅니다.
	- _Out_: 함수가 이 매개변수를 통해 값을 반환함을 나타냅니다.
	- _Inout_: 함수가 이 매개변수를 읽고 수정함을 나타냅니다.
2. 범위 및 크기
	- _In_reads_(size): 매개변수가 읽기 전용 배열이며, 배열의 크기가 size임을 나타냅니다.
	- _Out_writes_(size): 매개변수가 쓰기 전용 배열이며, 배열의 크기가 size임을 나타냅니다.
	- _In_opt_: 선택적으로 전달될 수 있는 매개변수(즉, NULL일 수 있음)를 나타냅니다.
3. 특수 조건
	- _Ret_maybenull_: 함수가 NULL 포인터를 반환할 수 있음을 나타냅니다.
	- _Success_(expr): 함수가 성공했을 때 특정 조건을 만족함을 나타냅니다.
